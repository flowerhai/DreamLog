## 🎨 截图内容规划

### 截图 1：首页 - 梦境列表 🏠

**展示内容**:
- 顶部统计卡片（连续记录天数/总梦境数）
- 梦境列表（3-4 个精美卡片）
- 底部导航栏（5 个 Tab）
- 搜索栏和筛选按钮

**文案设计**:
```
主标题：记录每个珍贵的梦境
副标题：支持文字、语音、图片多种记录方式
```

**设计要点**:
- 展示不同情绪的梦境卡片（多彩）
- 包含标签和情绪徽章
- 显示日期和时间
- 突出流畅的卡片设计
- 底部导航清晰可见

**拍摄步骤**:
1. 准备 3-4 个示例梦境（不同情绪/标签）
2. 确保统计卡片有数据
3. 使用模拟器或真机
4. 截取完整屏幕
5. 后期添加文案叠加

## 🛠️ 制作工具

### 截图工具

**Xcode Simulator**（推荐）
```
优点:
- 完美还原应用效果
- 支持所有设备尺寸
- 可以调整系统语言
- 支持深色/浅色模式

使用步骤:
1. 在 Xcode 中运行应用

3. 调整到目标界面
4. 文件 → 保存屏幕截图
5. 或使用 Cmd + S 快捷键
```

```

- 最真实的效果

- 测试实际性能

3. 选择设备 → View Device Logs

## 🎨 设计规范

### 文案样式

**主标题**:
- 字体：SF Pro Display Bold
- 字号：68pt
- 颜色：白色（#FFFFFF）
- 阴影：黑色 30% 不透明度，偏移 (0, 4)，模糊 8

**副标题**:
- 字体：SF Pro Text Regular
- 字号：42pt
- 颜色：白色 90% 不透明度（#FFFFFFE6）
- 阴影：黑色 20% 不透明度，偏移 (0, 2)，模糊 4

### 背景效果

**渐变背景**（推荐）:
```
顶部：#1a1a2e（深蓝）
中部：#16213e（中蓝）
底部：#0f3460（浅蓝）
```

**毛玻璃效果**:
```
背景模糊：20px
不透明度：60%
饱和度过：120%
```

### 安全区域

**顶部安全区**: 避开状态栏（约 100px）  
**底部安全区**: 避开 Home Indicator（约 80px）  
**左右安全区**: 各留 40px 边距

## 📐 截图模板

### 模板结构

```
┌─────────────────────────────┐
│                             │
│    [渐变背景/毛玻璃效果]     │
│                             │
│      主标题（大字号）        │
│                             │
│      副标题（小字号）        │
│                             │
│    ┌───────────────────┐    │
│    │                   │    │
│    │   [应用截图]       │    │
│    │                   │    │
│    │                   │    │
│    └───────────────────┘    │
│                             │
│    [装饰元素/图标]           │
│                             │
└─────────────────────────────┘
```

### 应用截图位置

**方案 A：居中**
- 截图垂直居中
- 文案在上方
- 简洁大气

**方案 B：偏下**
- 截图偏下方（占 60%）
- 文案在上方空白区
- 突出文案

**方案 C：倾斜**
- 截图轻微倾斜（5-10 度）
- 增加动感
- 适合 AR 功能展示

## 📤 导出设置

### 文件格式

**推荐**: PNG（无损压缩）  
**备选**: JPEG（高质量，85%+）  
**避免**: GIF/WebP（不支持）

### 导出参数

**PNG**:
- 颜色空间：sRGB
- 位深度：24 位
- 压缩：无
- 文件大小：~2-5MB/张

**JPEG**:
- 质量：90-95%
- 颜色空间：sRGB
- 文件大小：~500KB-1MB/张

### 命名规范

```
截图 1：dreamlog_home_6.7.png
截图 2：dreamlog_ai_6.7.png
截图 3：dreamlog_ar_6.7.png
截图 4：dreamlog_ml_6.7.png
截图 5：dreamlog_stats_6.7.png

截图 3：dreamlog_ar_6.1.png
截图 4：dreamlog_ml_6.1.png

## 🚀 快速制作流程

### 1 小时快速版

1. **准备数据**（10 分钟）
   - 创建 5-10 个示例梦境
   - 确保各功能有数据

2. **截取屏幕**（15 分钟）
   - 使用模拟器
   - 截取 5 个目标界面
   - 保存为 PNG

3. **添加文案**（20 分钟）
   - 使用 Figma 模板
   - 添加主副标题
   - 调整位置

4. **导出优化**（10 分钟）
   - 导出两种尺寸
   - 检查文件大小
   - 命名规范

5. **最终检查**（5 分钟）
   - 技术检查
   - 内容检查
   - 上传预览

### 专业版（4-6 小时）

1. **策划**（30 分钟）
   - 确定截图内容
   - 设计文案
   - 选择配色

2. **准备**（30 分钟）
   - 创建完美数据
   - 调整界面状态
   - 测试截图流程

3. **拍摄**（1 小时）
   - 使用真机拍摄
   - 多角度尝试
   - 选择最佳效果

4. **设计**（2-3 小时）
   - 精细后期处理
   - 添加特效
   - 多版本对比

5. **优化**（30 分钟）
   - 压缩优化
   - 多语言版本
   - 最终检查

## 📱 App Store Connect 上传

### 上传步骤

1. 登录 App Store Connect
2. 选择应用
3. 进入"版本"页面
4. 找到"App 预览和截图"
5. 选择设备类型（6.7 英寸）
6. 拖拽上传截图（按顺序）
7. 重复 6.1 英寸
8. 保存

### 注意事项

- 截图顺序很重要（第一张最重要）
- 可以重新排序
- 支持拖拽上传
- 支持批量上传
- 上传后预览效果

## 💡 最佳实践

### Do ✅

- 展示真实应用界面
- 使用精美文案
- 保持风格统一
- 突出核心功能
- 测试不同设备
- 考虑深色模式

### Don't ❌

- 使用模糊截图
- 文案过长
- 风格不统一
- 展示测试数据
- 包含敏感信息
- 违反 App Store 规范
