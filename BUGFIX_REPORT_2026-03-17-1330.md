## 🐛 发现并修复的问题

### 1. DreamChallengeService actor 与 SwiftUI 不兼容 🔴

**文件**: `DreamChallengeService.swift`  
**严重程度**: 高 (导致编译错误)  
**问题描述**: 
- `DreamChallengeService` 定义为 `actor`
- 在 `ContentView` 和 `DreamLogApp` 中使用 `@StateObject` 和 `@ObservedObject` 属性包装器
- SwiftUI 属性包装器要求 `ObservableObject` 协议，`actor` 不满足此要求
- 缺少 `shared` 单例，视图尝试使用 `DreamChallengeService.shared` 会导致编译错误
- 导致编译错误：`Property 'challengeService' with '@StateObject' wrapper must be an instance of a class that conforms to 'ObservableObject'`

**修复方案**:
```swift
// 修复前
actor DreamChallengeService {
    private let modelContext: ModelContext
    private let userId: String
    // ...
}

// 修复后
@MainActor
class DreamChallengeService: ObservableObject {
    static let shared = DreamChallengeService(modelContext: SharedModelContainer.main.context)
    
    @Published private let modelContext: ModelContext
    @Published private let userId: String
    @Published private var stats: ChallengeStats?
    // ...
}
```

**影响范围**:
- ContentView.swift (使用 @StateObject)
- DreamLogApp.swift (使用 @ObservedObject)
- DreamChallengeView
- 所有使用 `DreamChallengeService.shared` 的视图

**提交**: `46d6cff fix(concurrency): 修复 DreamChallengeService actor 与 SwiftUI 兼容性问题 🔧`
