**解决方案**: 使用 `guard let` 语句安全解包，失败时返回默认值

**修改内容**:
```swift
// 修复前
let yesterday = Calendar.current.date(byAdding: .day, value: -1, to: today)!

// 修复后
guard let yesterday = Calendar.current.date(byAdding: .day, value: -1, to: today) else {
    stats.currentStreak = 0
    stats.longestStreak = 0
    return
}
```

**影响**: 消除潜在崩溃风险，虽然 Calendar.date 在有效输入下很少失败

### 修复 4: DreamHealthIntegrationService - shared! 强制解包

**问题**: `initialize()` 方法中 `return shared!` 使用强制解包

**解决方案**: 使用 nil-coalescing 运算符提供回退

**修改内容**:
```swift
// 修复前
static func initialize(modelContainer: ModelContainer) -> DreamHealthIntegrationService {
    if shared == nil {
        shared = DreamHealthIntegrationService(modelContext: ModelContext(modelContainer))
    }
    return shared!
}

// 修复后
static func initialize(modelContainer: ModelContainer) -> DreamHealthIntegrationService {
    if shared == nil {
        shared = DreamHealthIntegrationService(modelContext: ModelContext(modelContainer))
    }
    return shared ?? DreamHealthIntegrationService(modelContext: ModelContext(modelContainer))
}
```

**影响**: 消除单例初始化失败时的崩溃风险
