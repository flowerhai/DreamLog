## 🔧 修复详情

### 修复 1: DreamPlaylistService.swift - Calendar 日期计算强制解包

**问题**: `Calendar.current.date(byAdding:...)` 使用 `!` 强制解包，可能导致运行时崩溃

**解决方案**: 使用 `??` 提供默认值

**修改内容**:
```swift
// 修复前
let descriptor = FetchDescriptor<Dream>(
    predicate: #Predicate { $0.date >= Calendar.current.date(byAdding: .day, value: -7, to: Date())! },
    ...
)

// 修复后
let sevenDaysAgo = Calendar.current.date(byAdding: .day, value: -7, to: Date()) ?? Date()
let descriptor = FetchDescriptor<Dream>(
    predicate: #Predicate { $0.date >= sevenDaysAgo },
    ...
)
```

**影响**: 避免在极端情况下（如日历计算失败）导致应用崩溃

---

### 修复 2: DreamMentionService.swift - Range 强制解包

**问题**: `Range(match.range, in: text)` 使用 `!` 强制解包

**解决方案**: 使用 `guard let` 安全解包

**修改内容**:
```swift
// 修复前
let fullRange = Range(match.range, in: text)!

// 修复后
guard let fullRange = Range(match.range, in: text) else { continue }
```

**影响**: 提高提及解析的健壮性，避免在边界情况下崩溃

---

### 修复 3: DreamShareHubEnhancedModels.swift - URL 强制解包 (3 处)

**问题**: 3 处 URL 初始化使用 `!` 强制解包

**解决方案**: 
- `canOpenURL` 检查：使用 `guard let` 安全解包
- `universalLink` 和 `fallbackLink`: 改为返回可选 URL

**修改内容**:
```swift
// 修复前
guard let scheme = scheme else { return false }
return UIApplication.shared.canOpenURL(URL(string: scheme)!)

// 修复后
guard let scheme = scheme, let url = URL(string: scheme) else { return false }
return UIApplication.shared.canOpenURL(url)

// 修复前
var universalLink: URL {
    URL(string: "https://dreamlog.app/d/\(shortCode)")!
}

// 修复后
var universalLink: URL? {
    URL(string: "https://dreamlog.app/d/\(shortCode)")
}
```

**影响**: 提高分享链接生成的健壮性
