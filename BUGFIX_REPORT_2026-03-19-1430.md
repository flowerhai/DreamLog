## 🔧 修复详情

### 1. DreamARShareService.swift

**问题**: MCNearbyServiceBrowserDelegate 中使用强制解包访问 session

**修复前**:
```swift
func browser(_ browser: MCNearbyServiceBrowser, foundPeer peerID: MCPeerID, withDiscoveryInfo info: [String: String]?) {
    print("🔍 发现主持者：\(peerID.displayName)")
    browser.invitePeer(peerID, to: session!, withContext: nil, timeout: 30)
}
```

**修复后**:
```swift
func browser(_ browser: MCNearbyServiceBrowser, foundPeer peerID: MCPeerID, withDiscoveryInfo info: [String: String]?) {
    print("🔍 发现主持者：\(peerID.displayName)")
    guard let session = session else {
        print("❌ Session 未初始化，无法邀请 peer")
        return
    }
    browser.invitePeer(peerID, to: session, withContext: nil, timeout: 30)
}
```

**影响**: 改进 MultipeerConnectivity 会话安全性
**风险**: 低（session 通常在 init 中初始化，但安全处理更佳）

### 2. DreamChallengeService.swift

**问题**: SwiftData Predicate 中使用强制解包访问 startedAt

**修复前**:
```swift
let descriptor = FetchDescriptor<DreamChallenge>(
    predicate: #Predicate<DreamChallenge> { challenge in
        challenge.startedAt != nil &&
        challenge.startedAt! >= startOfDay &&
        challenge.startedAt! < endOfDay
    }
)
```

**修复后**:
```swift
let descriptor = FetchDescriptor<DreamChallenge>(
    predicate: #Predicate<DreamChallenge> { challenge in
        if let startedAt = challenge.startedAt {
            return startedAt >= startOfDay && startedAt < endOfDay
        }
        return false
    }
)
```

**影响**: 改进 SwiftData 查询安全性
**风险**: 无（逻辑等价，但更安全）
