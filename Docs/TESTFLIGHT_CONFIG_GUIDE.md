## 📋 TestFlight 概述

TestFlight 是 Apple 官方的应用测试平台，允许开发者在 App Store 发布前邀请用户测试应用。

**测试优势**:
- 真实设备测试
- 快速迭代更新
- 收集用户反馈
- 发现潜在问题
- 验证功能完整性

---

## 🎯 测试阶段规划

### 阶段 1: 内部测试 (2 天)

**目标**: 核心功能稳定性验证

**测试人员**: 10-20 人
- 开发团队成员
- 亲密朋友/家人
- 信任的 Beta 测试者

**测试重点**:
- [ ] 应用能否正常启动
- [ ] 核心功能是否可用
- [ ] 是否存在崩溃
- [ ] 数据保存是否正常
- [ ] iCloud 同步是否工作

**通过标准**:
- 无 P0/P1 级别 Bug
- 崩溃率 < 1%
- 核心功能 100% 可用

### 阶段 2: 外部测试 (5 天)

**目标**: 功能完整性与用户体验验证

**测试人员**: 100-500 人
- 公开招募的测试者
- 梦境爱好者社区
- 产品目标用户群体

**测试重点**:
- [ ] 所有功能正常使用
- [ ] 用户体验流畅
- [ ] 性能表现良好
- [ ] 兼容性 (不同设备/iOS 版本)
- [ ] 用户反馈收集

**通过标准**:
- 无 P0/P1 级别 Bug
- 崩溃率 < 0.5%
- 用户满意度 > 4.0/5.0
- 核心功能使用率 > 80%

## ⚙️ TestFlight 配置步骤

### 步骤 1: App Store Connect 配置

1. **登录 App Store Connect**
   ```
   https://appstoreconnect.apple.com
   ```

2. **进入 TestFlight 标签页**
   - 选择 DreamLog 应用
   - 点击顶部 "TestFlight" 标签

3. **添加测试人员信息**
   - 点击 "App 信息"
   - 填写测试联系邮箱
   - 填写测试说明 (可选)

### 步骤 2: 创建内部测试组

1. **点击 "内部测试员"**
2. **点击 "+" 创建新组**
3. **组名**: `DreamLog Internal`
4. **添加成员**:
   - 输入测试人员邮箱
   - 或从用户列表选择
5. **保存**

### 步骤 3: 创建外部测试组

1. **点击 "外部测试员"**
2. **点击 "+" 创建新组**
3. **组名**: `DreamLog Beta`
4. **设置测试人数限制**: 500
5. **配置测试信息**:
   - 测试说明 (中文 + 英文)
   - 反馈邮箱
6. **保存**

### 步骤 4: 上传构建版本

1. **在 Xcode 中归档**
   ```bash
   # 选择 Generic iOS Device 或真机
   # Product → Archive
   ```

2. **上传到 App Store Connect**
   - Archive 完成后自动弹出 Organizer
   - 点击 "Distribute App"
   - 选择 "App Store Connect"
   - 选择 "Upload"
   - 完成上传流程

3. **等待处理完成**
   - 通常 10-30 分钟
   - 状态变为 "准备提交" 或 "正在测试"

### 步骤 5: 提交 Beta 审核

**外部测试需要 Apple 审核**:

1. **选择构建版本**
   - 在 TestFlight 页面选择刚上传的版本

2. **填写测试信息**
   - 测试目的说明
   - 测试重点功能
   - 已知问题说明

3. **提交审核**
   - 点击 "提交审核"
   - 等待审核结果 (通常 24-48 小时)

4. **审核通过后**
   - 构建版本状态变为 "正在测试"
   - 可以邀请测试人员

### 步骤 6: 邀请测试人员

**内部测试**:
- 自动获得邀请邮件
- 无需审核，立即可用

**外部测试**:
1. **点击测试组**
2. **添加测试人员**:
   - 手动输入邮箱
   - 或导入 CSV 文件
3. **发送邀请**
4. **测试人员收到邮件**
   - 点击链接下载 TestFlight
   - 安装 DreamLog Beta

### 步骤 7: 公开测试链接 (可选)

1. **在外部测试组页面**
2. **点击 "公开测试链接"**
3. **复制链接**
4. **分享给测试人员**

**公开链接特点**:
- 任何人点击都可加入测试
- 最多 10,000 名测试者
- 无需手动添加邮箱

## 🐛 Bug 分级与响应

### Bug 分级标准

**P0 - 严重 (Critical)**
- 应用无法启动
- 数据丢失
- 严重崩溃 (>10% 用户)
- 安全漏洞

**响应时间**: 立即修复，24 小时内发布更新

**P1 - 高 (High)**
- 核心功能不可用
- 频繁崩溃 (1-10% 用户)
- 数据同步失败
- 主要 UI 问题

**响应时间**: 48 小时内修复

**P2 - 中 (Medium)**
- 非核心功能问题
- 偶发崩溃 (<1% 用户)
- UI 小问题
- 性能问题

**响应时间**: 下次更新修复

**P3 - 低 (Low)**
- 轻微 UI 问题
- 文案错误
- 建议性改进

**响应时间**: 视优先级安排
