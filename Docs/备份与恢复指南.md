# 备份与恢复使用指南

> Phase 29 - 梦境备份与恢复系统增强

---

## 📖 概述

DreamLog 提供完整的本地备份与恢复功能，帮助您保护珍贵的梦境数据。支持 AES-256 加密、自动备份计划和选择性备份。

---

## 🗄️ 创建备份

### 步骤

1. **打开设置**
   - 在应用底部导航栏点击"设置"
   - 找到"数据与同步"部分
   - 点击"🗄️ 备份与恢复"

2. **配置备份选项**
   
   在"创建备份"标签页，您可以选择：
   
   - **备份范围**
     - ✅ 备份所有梦境
     - ⬜ 按日期范围备份（选择开始和结束日期）
   
   - **内容选项**
     - ✅ 包含音频录音
     - ✅ 包含梦境图片
   
   - **安全选项**
     - ✅ 加密备份文件（推荐）
     - 设置备份密码（如果启用加密）

3. **开始备份**
   - 点击"创建备份"按钮
   - 等待备份完成（显示进度）
   - 查看备份结果（梦境数量、文件大小）

### 备份选项说明

| 选项 | 说明 | 推荐设置 |
|------|------|----------|
| 备份所有梦境 | 备份全部梦境数据 | ✅ 启用 |
| 包含音频 | 包含梦境录音文件 | ✅ 启用（需要更多空间） |
| 包含图片 | 包含 AI 生成的梦境图片 | ✅ 启用（需要更多空间） |
| 加密备份 | 使用 AES-256 加密 | ✅ 强烈推荐 |

### 备份文件格式

- **文件扩展名**: `.dreamlog`
- **格式**: JSON（加密后为二进制）
- **包含内容**:
  - 梦境文本数据
  - 标签和情绪
  - 音频文件（可选）
  - 图片文件（可选）
  - 元数据（设备信息、备份时间等）

## 🔄 恢复备份

### 步骤

1. **选择备份文件**
   - 切换到"恢复备份"标签页
   - 点击"选择备份文件"
   - 从文件选择器中找到 `.dreamlog` 文件

2. **输入密码（如果加密）**
   - 如果备份已加密，系统会提示输入密码
   - 输入创建备份时设置的密码

3. **确认恢复**
   - 系统会显示确认对话框
   - 点击"恢复"开始导入数据

4. **等待完成**
   - 查看恢复进度
   - 恢复完成后查看结果

### 重复检测

恢复时会自动检测重复的梦境：

- ✅ **跳过重复** - 如果梦境 ID 已存在，自动跳过
- 📊 **显示统计** - 恢复完成后显示恢复数量和跳过数量

### 恢复选项

| 选项 | 说明 |
|------|------|
| 跳过重复 | 如果梦境已存在，不覆盖（推荐） |
| 覆盖现有 | 用备份数据覆盖现有梦境（谨慎使用） |

## ⚙️ 自动备份

### 设置自动备份

1. 切换到"自动备份"标签页
2. 启用"启用自动备份"
3. 配置以下选项：
   - **备份频率**: 每日/每周/每月
   - **备份时间**: 选择具体时间（如 02:00）
   - **保留数量**: 保留最近 N 个备份（默认 5 个）
4. 点击"保存设置"

### 自动备份说明

- **执行时间**: 在设定的时间自动创建备份
- **加密**: 自动备份默认启用加密
- **清理**: 超过保留数量的旧备份自动删除
- **位置**: 备份保存在 `Documents/DreamLogBackups/`

### 推荐设置

| 使用场景 | 频率 | 时间 | 保留数量 |
|----------|------|------|----------|
| 轻度用户 | 每周 | 周日 02:00 | 4 个 |
| 活跃用户 | 每日 | 02:00 | 7 个 |
| 重度用户 | 每日 | 02:00 | 14 个 |

## 🔒 安全建议

### 密码管理

1. **使用强密码**
   - 至少 8 个字符
   - 包含大小写字母、数字、符号
   - 避免使用常见单词

2. **保存密码**
   - 将密码保存在密码管理器中
   - 不要将密码与备份文件放在一起
   - 记住：丢失密码 = 无法恢复备份

3. **定期更换**
   - 建议每 3-6 个月更换一次密码
   - 更换密码后创建新备份

### 备份存储

1. **多地点存储**
   - 保留本地备份
   - 定期导出到电脑
   - 使用云存储（iCloud Drive、Dropbox 等）

2. **定期验证**
   - 定期检查备份文件是否可恢复
   - 验证备份完整性

3. **版本管理**
   - 保留多个历史版本
   - 重要更新前手动备份

## ❓ 常见问题

### Q: 备份文件保存在哪里？

A: 备份文件保存在应用的文档目录：
```
Documents/DreamLogBackups/
```

您可以通过文件 App 访问这个目录。

### Q: 备份文件有多大？

A: 备份大小取决于：
- 梦境数量
- 是否包含音频（音频文件较大）
- 是否包含图片

示例：
- 100 条纯文本梦境：~500 KB
- 100 条梦境 + 音频：~50 MB
- 100 条梦境 + 音频 + 图片：~100 MB

### Q: 加密会影响性能吗？

A: 轻微影响。加密/解密过程会增加几秒钟时间，但为了安全性，这是值得的。

### Q: 可以恢复单个梦境吗？

A: 当前版本不支持。恢复操作会导入备份中的所有梦境（跳过重复的）。

### Q: 备份会包含删除的梦境吗？

A: 不会。备份只包含当前存在的梦境。

### Q: 自动备份失败怎么办？

A: 检查以下事项：
1. 设备是否有足够存储空间
2. 应用是否有文件访问权限
3. 备份密码是否正确设置
4. 查看备份历史了解错误信息

## 📊 技术细节

### 加密算法

- **算法**: AES-256-GCM
- **密钥派生**: PBKDF2
- **认证**: GCM 模式提供完整性保护

### 数据格式

```json
{
  "metadata": {
    "version": "1.0",
    "appVersion": "1.0.0",
    "backupDate": "2026-03-12T20:00:00Z",
    "deviceName": "iPhone",
    "dreamCount": 100,
    "encryptionMethod": "AES-256-GCM",
    "checksum": "abc123..."
  },
  "dreams": [...],
  "tags": [...],
  "audioFiles": {...},
  "imageFiles": {...}
}
```

### 完整性校验

- **算法**: SHA1
- **用途**: 验证备份文件完整性
- **存储**: 保存在备份历史和元数据中
