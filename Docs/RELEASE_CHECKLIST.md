## 🎯 发布策略

### 阶段 1: TestFlight 内部测试（1 周）

- **测试人员**: 开发团队 + 亲友（10-20 人）
- **目标**: 发现严重 bug
- **重点**: 核心功能稳定性

### 阶段 2: TestFlight 外部测试（2 周）

- **测试人员**: 公开招募（100-500 人）
- **目标**: 收集用户体验反馈
- **重点**: 功能完整性和易用性

### 阶段 3: 软发布（1 周）

- **地区**: 新西兰/新加坡/加拿大（小市场）
- **目标**: 验证 App Store 审核流程
- **重点**: 监控崩溃率和用户反馈

### 阶段 4: 全球发布

- **地区**: 全球 175+ 国家/地区
- **时间**: 周二或周三（最佳发布日）
- **营销**: 社交媒体 + Product Hunt + 应用评测
